# Adorn

Generic wearable item system — slot management, stat aggregation, visual attachment, and set bonuses.

| | |
|---|---|
| **Category** | Gameplay |
| **UE Version** | 5.5+ |
| **Blueprint API** | Yes |
| **C++ API** | Yes |
| **GAS Required** | No |
| **Replication** | Yes |

Adorn handles the full lifecycle of equippable wearable items: Data Asset-driven item definitions, a slot system (head, chest, legs, hands, feet, ring, neck, etc.), automatic stat aggregation across all worn items, visual mesh swapping or attachment, and set bonuses when multiple items from the same set are worn. Unlike Arsenal (which is weapon/equipment focused), Adorn is designed for RPG-style character customization and progression.
